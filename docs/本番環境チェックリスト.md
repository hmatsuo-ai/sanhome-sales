# 本番環境デプロイ チェックリスト

本番環境にデプロイする前に確認・実施すべき項目です。

## 必須

- [ ] **AUTH_SECRET**  
  - 強力なランダム文字列を設定（例: `openssl rand -base64 32`）
  - 本番では必ず開発環境と異なる値にする

- [ ] **DATABASE_URL**  
  - 本番用DB（PostgreSQL 推奨）のURLを設定
  - SQLite は本番では非推奨（同時アクセス・バックアップの観点）

- [ ] **NEXT_PUBLIC_APP_URL**  
  - 本番のURL（`https://your-domain.com`）を設定

- [ ] **HTTPS**  
  - リバースプロキシまたはホスティングで HTTPS を有効化

- [ ] **認証プロキシ**  
  - `src/proxy.ts` により未ログイン時は `/login` へリダイレクトされます（有効であることを確認）

- [ ] **マイグレーション**  
  - デプロイ時に `npx prisma migrate deploy` を実行（CI/CD に組み込む）

## 推奨

- [ ] **API 認可**  
  - 全 API ルートで `auth()` を呼び、必要に応じてロール（admin 等）をチェック
  - アップロード・OCR API は認証必須＋レート制限を検討

- [ ] **アップロード**  
  - `/api/upload` に認証を追加
  - 許可する MIME タイプ・最大サイズを制限
  - 本番ではストレージを S3 等に変更することを検討

- [ ] **エラーハンドリング**  
  - `app/error.tsx` と `app/not-found.tsx` でユーザー向けエラー表示を実施済み
  - 本番では `NODE_ENV=production` で内部エラーをログにのみ出力することを確認

- [ ] **セキュリティヘッダー**  
  - `next.config` またはミドルウェアで Content-Security-Policy, X-Frame-Options 等を設定

- [ ] **レート制限**  
  - ログイン・フィードバック・アップロード・OCR にレート制限を検討

## 環境変数一覧（.env.example 参照）

| 変数名 | 必須 | 説明 |
|--------|------|------|
| DATABASE_URL | ○ | データベース接続URL |
| AUTH_SECRET | ○ | NextAuth 用秘密鍵 |
| NEXT_PUBLIC_APP_URL | ○ | アプリの公開URL |
| GEMINI_API_KEY | - | 領収書OCR用（任意） |

## デプロイ後の確認

- 未ログインで `/dashboard` 等にアクセス → `/login` にリダイレクトされること
- ログイン後、ダッシュボード・売上・経費・設定が表示されること
- 存在しないURL → 404 ページが表示されること
- エラー発生時 → エラーページが表示され、再試行・ダッシュボードへ戻れること
