# ローカル環境での動作確認

このドキュメントでは、PC 上でサンホーム営業管理システムを動かす手順を説明します。

---

## 前提条件

- **Node.js**: v18 以上（推奨: v20 以上）
- **npm**: Node.js に同梱されています

ターミナルで確認:
```bash
node -v   # v18.x または v20.x など
npm -v
```

---

## 手順 1: リポジトリのディレクトリへ移動

```bash
cd "c:\Users\Owner\VEXUM\サンホーム\営業担当者管理システム\sanhome-sales"
```

（または、ターミナルで `sanhome-sales` フォルダを開いた状態で以下を実行）

## 手順 2: 環境変数（.env）の用意

プロジェクト直下に **`.env`** ファイルを作成し、次の内容を設定します。

### SQLite で動かす場合（初期設定・手軽）

```env
DATABASE_URL="file:./dev.db"
AUTH_SECRET="任意の長いランダム文字列"
```

- `AUTH_SECRET` はログインのセッション暗号化用です。本番では `openssl rand -base64 32` などで生成した値を使うことを推奨します。ローカル確認だけなら任意の英数字で構いません。
- データベースファイルは `prisma/dev.db` に自動作成されます。

### （参考）PostgreSQL で動かす場合

PostgreSQL をインストールし、DB を作成したうえで:

```env
DATABASE_URL="postgresql://ユーザー名:パスワード@localhost:5432/データベース名"
AUTH_SECRET="任意の長いランダム文字列"
```

その場合、`prisma/schema.prisma` の `provider` を `"postgresql"` に変更する必要があります。

## 手順 5: 開発サーバーの起動

```bash
npm run dev
```

次のような表示が出れば起動成功です:

```
▲ Next.js 16.x.x
- Local:        http://localhost:3000
```

**Windows で Turbopack のエラーが出る場合**（Prisma の junction 関連など）は、次のいずれかで起動してください:

```bash
npm run dev:no-turbo
```

または:

```bash
npx next dev --no-turbopack
```

## 手順 6: ブラウザで確認

1. ブラウザで **http://localhost:3000** を開く
2. ログイン画面が表示される
3. 上記のいずれかのメールアドレスとパスワード `password123` でログイン
4. ダッシュボード・経費精算・売上管理・スケジュール・設定（管理者のみ）が利用できることを確認

## よくあるトラブル

| 現象 | 対処 |
|------|------|
| `DATABASE_URL` がない | プロジェクト直下に `.env` を作成し、`DATABASE_URL` と `AUTH_SECRET` を設定する |
| `prisma migrate` でエラー | SQLite のときは `DATABASE_URL="file:./dev.db"` になっているか確認。PostgreSQL のときは DB が起動しているか・URL が正しいか確認 |
| ログインできない | `npm run db:seed` を実行したか確認。メールとパスワードは `tanaka@sanhome.co.jp` / `password123` など |
| ポート 3000 が使われている | `npm run dev -- -p 3001` のように別ポートで起動できる |
| `npm install` で peer dependency エラー | Next.js 16 と next-auth の競合の場合がある。`npm install --legacy-peer-deps` でインストールする |
| Turbopack で Prisma のエラー（Windows） | `npx next dev --no-turbopack` で起動する |
---

## コマンド一覧（まとめ）

```bash
cd sanhome-sales
# .env を用意（上記参照）
npm install
# peer dependency エラーが出る場合は: npm install --legacy-peer-deps
npm run db:generate
npm run db:migrate
npm run db:seed
npm run dev
# Windows で Turbopack エラーが出る場合は: npx next dev --no-turbopack
```

その後、ブラウザで http://localhost:3000 を開き、ログインして動作を確認してください。
