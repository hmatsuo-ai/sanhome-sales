# サンホーム 営業統合管理システム — ユーザー向け仕様書

## 1. 概要

**サンホーム 営業統合管理システム**は、営業担当者の売上・経費・スケジュールを一括で管理するWebアプリケーションです。  
ログイン後、ダッシュボード・経費精算・売上管理・スケジュール・設定の各画面から業務を行います。

---

## 2. ログイン・認証

### 2.1 ログイン方法

- **URL**: アプリのログイン画面にアクセスします。
- **入力項目**
  - **メールアドレス**: 登録済みのメールアドレス（例: example@sunhome.co.jp）
  - **パスワード**: 6文字以上のパスワード
- **操作**: 「ログインする」をクリックして認証します。
- メールアドレスまたはパスワードが誤っている場合、画面上にエラーメッセージが表示されます。

### 2.2 ログアウト

- 画面左のサイドバー（またはモバイルではメニュー内）の「ログアウト」をクリックするとログアウトし、ログイン画面に戻ります。

### 2.3 権限の種類

| 権限 | 説明 | 利用できる機能 |
|------|------|----------------|
| **営業** | 一般の営業担当者 | ダッシュボード、経費精算、売上管理、スケジュール |
| **管理者** | システム管理者 | 上記に加え **設定**（ユーザー・グループ管理）にアクセス可能 |

## 3. 画面構成と機能

### 3.1 ダッシュボード

**目的**: 売上・粗利・経費の集計と、本日のスケジュールを一覧で確認します。

- **表示切替**
  - **表示**: 「月別」「年別」を選択できます。
  - **期間**: 月別の場合は年月、年別の場合は年を選択します。
  - **担当者**: 「全員」または特定の担当者を選択して、その範囲で集計を表示します。
- **表示内容**
  - **今月の売上合計**: 選択期間の売上合計（粗利の内訳付き）
  - **今月の粗利合計**: 選択期間の粗利合計（粗利率表示あり）
  - **今月の経費合計**: 選択期間の経費合計
  - **本日のスケジュール**: 当日の予定一覧（タイトル・時間・場所）

※ラベルは「今月」ですが、選択した期間（月別/年別）に応じて集計されます。

---

### 3.2 経費精算

**目的**: 経費の登録・一覧表示・月別集計を行います。

- **期間**: 画面上部の月選択で、対象月を変更できます。
- **新規登録**
  - 「新規登録」ボタンで経費登録モーダルを開きます。
  - **入力項目**
    - **日付**: 経費が発生した日付
    - **カテゴリ**: 飲食費・交通費・駐車場代・接待費・消耗品費・宿泊費・通信費・書籍・資料・その他
    - **金額（円）**: 税込金額
    - **メモ**: 店名・用途など（任意）
    - **領収書画像（任意）**: 画像を選択またはドラッグ＆ドロップでアップロード可能
  - 「保存」で登録します。
- **一覧**
  - 日付・担当者・カテゴリ・金額・メモ・領収書（画像リンクありの場合）を表示します。
  - 領収書画像がある場合は「画像」リンクから別タブで表示できます。
- **個人別サマリー**
  - ドロップダウンで担当者を選ぶと、その担当者の月間経費合計と件数を表示します。
  - 「詳細を表示」リンクから、その担当者の詳細ページ（ユーザー詳細）に遷移できます。
- **削除**
  - 自分が登録した経費のみ「削除」ボタンで削除できます。

---

### 3.3 売上管理

**目的**: 売上・粗利の登録、複数担当者への配分、決済状況の管理を行います。

- **タブ**
  - **事業利益・仲介**: カテゴリ「事業利益」「仲介」の売上を表示・集計します。
  - **賃貸（いい部屋）**: カテゴリ「いい部屋ネット」の売上を表示・集計します。
- **期間**
  - 開始日・終了日で表示対象期間を指定できます。
  - 初期表示は半期（4月〜9月 / 10月〜翌3月）です。
- **新規登録**
  - 「新規登録」で売上登録モーダルを開きます。
  - **入力項目**
    - **契約日**
    - **決済日（必須）**
    - **担当者と配分比率（%）**: 複数担当者を選択し、各担当者の粗利配分比率を%で指定。未指定・合計が100%でない場合は均等配分されます。
    - **カテゴリ**: いい部屋ネット / 仲介 / 事業利益
    - **案件名**
    - **売上金額**・**粗利金額**
  - 「保存」で登録します。担当者は1名以上選択必須です。
- **一覧**
  - 契約日・担当者・案件名・カテゴリ（事業利益・仲介タブ時）・売上(総額)・粗利(総額)・決済日・決済状況・操作 を表示します。
  - **未決済のみ表示**: チェックを入れると、未決済の売上だけを表示します。
  - **決済状況**: 「未決済」「決済済」ボタンでトグルできます。
  - **削除**: ゴミ箱アイコンで当該売上を削除できます（確認ダイアログあり）。
- **サマリー**
  - **全体サマリー**: 選択期間の全体売上・全体粗利を表示します。
  - **個人別サマリー**: 担当者を選択すると、選択期間の粗利（配分後）と半期通算の粗利（配分後）を表示します。
  - 「詳細を表示」から、その担当者のユーザー詳細ページへ遷移できます。

---

### 3.4 スケジュール

**目的**: 予定の登録・一覧表示を、全員・グループ・個人の3つの見方で行います。

- **表示モード**
  - **全員**: 全担当者の予定を週間カレンダーで表示します。
  - **グループ**: グループを選択し、そのグループ所属メンバーを列にして1日分の予定を表示します。
  - **個人**: ユーザーを選択し、その担当者の週間カレンダーを表示します。
- **週・日の移動**
  - 「＜」「今日」「＞」で、週（全員/個人）または日（グループ）を前後に移動できます。
- **予定の追加**
  - 「追加」ボタン、またはカレンダーの空き枠をクリックして登録モーダルを開きます。
  - **入力項目**: タイトル・日付・開始時刻・終了時刻・場所（任意）
  - 「保存」で登録します。タイトルは必須です。
- **予定の編集・削除**
  - 予定をクリックすると編集モーダルが開き、内容を変更して保存できます。
  - 自分が登録した予定には「✕」ボタンが表示され、クリックで削除できます（確認あり）。

---

### 3.5 設定（管理者のみ）

**目的**: ユーザー・グループの登録・編集・削除を行います。

- **新規ユーザー登録**
  - 名前・メールアドレス・初期パスワード・権限（営業担当 / 管理者）を入力し、「ユーザーを登録」で登録します。
- **グループ管理**
  - 新グループ名を入力して「追加」でグループを作成します。
  - 一覧の削除アイコンでグループを削除できます。削除時、所属ユーザーのグループは「未設定」になります。
- **ユーザー一覧・所属設定**
  - 各ユーザーの名前・メール・権限を表示します。
  - 「所属グループ」でグループを選択すると、そのユーザーをそのグループに紐付けます。
  - 「個人詳細・パスワード変更」リンクから、ユーザー詳細ページ（`/users/[id]`）に遷移できます。

---

### 3.6 ユーザー詳細（個人詳細・パスワード変更）

**目的**: 特定の担当者の売上・経費・スケジュールの集計と、アカウント設定（パスワード変更）を行います。

- **アクセス**
  - 設定画面の「個人詳細・パスワード変更」、または売上管理・経費精算の「詳細を表示」から遷移できます。
- **表示内容**
  - **期間粗利（配分後）**: 日付範囲を指定して、その担当者に配分された粗利の合計を表示します。
  - **月間経費合計**: 月を選択し、その月の経費合計を表示します。
  - **今週の予定数**: 今週のスケジュール件数を表示します。
  - **経費詳細**: 選択月の経費一覧（日付・カテゴリ・金額・メモ）。
  - **売上実績（案件別配分）**: 選択期間の売上一覧と、その担当者に配分された粗利（割合付き）。
  - **週間スケジュール**: 今週の日別予定一覧。
- **アカウント設定**
  - 「新しいパスワード」を入力し「更新」で、そのユーザーのパスワードを変更できます。本人または管理者が利用できます。

## 4. データの考え方（ユーザー向け説明）

- **売上**
  - 1件の売上に複数担当者を紐付けでき、各担当者に粗利配分比率（%）を指定できます。指定がない・合計が100%でない場合は均等に配分されます。
  - 「決済日」は必須です。「決済済」フラグで未決済・決済済を管理できます。
- **経費**
  - 経費は登録した担当者に紐付きます。自分が登録した経費のみ削除できます。
  - 領収書画像は任意で添付でき、一覧から画像を開いて確認できます。
- **スケジュール**
  - 予定は登録した担当者に紐付きます。全員表示・グループ表示・個人表示で参照できます。
- **グループ**
  - 担当者をグループに所属させることができます。スケジュールの「グループ」表示で、グループ単位の予定を確認できます。

## 5. 利用環境・注意事項

- **推奨環境**: 現代のブラウザ（Chrome、Edge、Safari、Firefoxなど）で利用してください。
- **モバイル**: スマートフォンでも利用可能です。画面下部にナビゲーションが表示されます。
- **パスワード**: 初期パスワードは設定画面で管理者が設定します。本人はユーザー詳細からパスワードを変更することを推奨します。
- **削除操作**: 売上・経費・スケジュール・グループの削除時は確認メッセージが表示されます。実行前に内容を確認してください。

## 7. 動作環境（サーバー・実行環境）

### 想定している実行環境

このアプリは **Node.js** 上で動く **Next.js** アプリケーションです。次のような環境で動作することを想定しています。

| 環境 | 説明 |
|------|------|
| **ローカル開発** | `npm run dev` で PC 上で Next.js 開発サーバーを起動して利用 |
| **Vercel** | Next.js に最適なホスティング。デプロイすればそのまま利用可能 |
| **Railway / Render / その他 PaaS** | Node.js 対応のクラウドで `npm run build` → `npm run start` で稼働 |
| **自前サーバー（VPS 等）** | Node.js をインストールし、ビルド後に `next start` で常時起動 |

**技術的な前提**

- **ランタイム**: Node.js（npm でパッケージをインストールして実行）
- **データベース**: Prisma 経由で SQLite（または PostgreSQL 等に変更可能）
- **認証**: NextAuth（サーバー側セッション）
- **その他**: ファイルアップロード、API ルート、サーバーコンポーネントなど、Node.js サーバーが必要な機能を使用

このため、**ブラウザだけで完結する静的サイトではなく、常時起動しているサーバー（またはサーバーレス関数）が必要**です。

### Google Apps Script（GAS）で動かせるか

**結論: このアプリをそのまま GAS で動かすことはできません。**

理由は次のとおりです。

1. **ランタイムが違う**  
   GAS は Node.js ではなく、Google が提供する JavaScript 実行環境です。`npm` や `node_modules` は使えません。

2. **Next.js / Prisma が使えない**  
   Next.js・Prisma・NextAuth・bcrypt など、Node.js 用のライブラリは GAS では利用できません。

3. **アーキテクチャが違う**  
   このアプリは「Next.js サーバー ＋ データベース」の構成です。GAS は「doGet/doPost の Web アプリ」や「スプレッドシート連携」などの用途向けで、同じ形で置き換えることはできません。

**GAS を活用したい場合の選択肢**

- **このアプリは今のまま Node.js 環境で運用**し、GAS からは「連携用の補助」として使う  
  - 例: スプレッドシートと連携する場合は、GAS でスプレッドシートを読み書きし、このアプリの API（`/api/sales` など）を GAS の `UrlFetchApp` で呼び出す。
- **GAS だけで似た機能を作る**場合は、GAS 用に**別アプリとして一から実装**する必要があります（スプレッドシートを DB 代わりにする、GAS Web アプリで簡易 UI を作る、など）。
